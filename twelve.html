Ruby Notes 12

We All Deserve Arrays

a = [1, 2, 3, 4, 5, 6, 7, 8, 9]

Call an array element with a negative number to count from the end.
a[-1] returns 9.
Pairs of comma-separated numbers call segments:
a[1,3] returns 2,3,4
where 1 is the start-point and 3 is the number of elements.
Pairs of dot-separated numbers call ranges:
a[1..3] and a[1...4] both return 2,3,4
where two-dot includes the end-point and three-dot excludes it.
Write Array Methods into Class Definition

class SongList

  def initialize

    @songs = Array.new

  end

  def appned aSong

    @songs.push(aSong)

    self

  end

  def deleteFirst

    @songs.shift

  end

  def deleteLast

    @songs.pop

  end

end

Now, our array methods are, for demo purposes, shorthanded to methods in our SongList class definition. Remember the Song class? Here it comes:

list1 = SongList.new

list1.append(Song.new("Panasonic Youth","DEP","2:33"))

list1.append(Song.new("Phthisis","SGM","3:46"))

Notice how we can create new Song objects at the moment we append them to the SongList object.

Give anybody any design ideas?

Expanding the Access-Elements-by-Index Method

This adds two array access techniques - index number and value - within a single method call

class SongList

  def [] (key)

    if key.kind_of?(Integer)

      result = @songs[key]

    else

      result = @songs.find {|aSong| key == aSong.name}

    end

    return result

  end

end

Now, whether we type the index number or the song name between square brackets the [] method returns our array element.

The .find method is an iterator, applying a test to each array element. It's the Ruby equivalent of writing:

for i in 0...@songs.length

  return @songs[i] if key == @songs[i].name

end

Using conditionality shorthand, we can shorten the [] method to two lines.

class SongList

  def [] (key)

     return @songs[key] if key.kind_of?(Integer)

     return @songs.find {|aSong| key == aSong.name}

  end

end
