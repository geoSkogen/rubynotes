String Literals - Single vs. Double Quotes

Ruby does very little to process strings between single quotes. Strings between double quotes have two useful features, however.

substitution sensitivity:
Ruby will parse typographical characters like \n (new line) within double-quoted strings.
expression interpolation:
Ruby reads variables and expressions within strings, without breaking the quotes:
puts "sometext #{var1Name} sometext."

The #{} configuration accepts arbitrarily complex expressions. When simply interpolating global vars or class vars, no brackets are required.

Variable Naming Rules - First Character

local variables, methods - and their parameters:
_mylocal  do_meth  param1  - lowercase or underscore
classes, modules, and constants:
MyObject   - uppercase letter only!
globals:
$myGlobal
instance variables:
@thisInstance
class variables:
@@classVar

Shorthand!

Just in case you didn't want to assume:

a1 = [] is the same as a2 = Array.new
- and the same goes for hashes.

Populate an Array with Strings Shorthand!

a = %w { str1 str2 str3 } is the same as

a = {"str1","str2","str3"}.

Conditionality Shorthand!

puts "Danger" if x > 3000 is equal to
