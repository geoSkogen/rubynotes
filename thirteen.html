Ruby Notes 13

Blocks & Iterators

A code block between braces { } may appear in the source adjacent to a method call, on the same line as the method's last param.

Ruby remembers the block's context, enters the called method, then executes the block as if it were a method itself.

yield invokes the code block within the method.

def doItTwice

  yield

  yield

end


doItTwice { puts "Hello Ruby"}

We can pass params to blocks as well, and recieve values back. Declare the arguments between pipes precedin the code block

def fibUpTo(max)

  i1, i2 = 1, 1    #parallel assignment

  while i1 <= max

    yield i1    #this passes i1 through the code block via f

    i1, i2 = i2, i1 + i2

  end

end


fibUpTo(1000) {|f| print f, " "} #declare arguments between pipes

The .find method, for example, is an array iterator which takes its arguments in block form.

a = [1, 2, 3, 4, 5, 6, 7, 8, 9]

a.find {|v| v*v > 30}

Returns the first element to fulfill the condition, a number whose square is greater than thirty.
